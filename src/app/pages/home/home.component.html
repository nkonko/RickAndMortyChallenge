Welcome to Rick & Morty Angular App!

The objective of the exercise is to create an application from scratch using Angular in its latest version. We will use a public API for the exercises: https://rickandmortyapi.com/documentation. Reusing components as well as clarity in the code and tests will be positively evaluated. It's important to highlight that the application must be responsive for both PC and tablet devices. No specific dimensions are specified. Please, reply to this mail with the code in a zip file. I hope you enjoy the test.

Exercise 1: Create an Angular project from scratch.

The main view will be the entry point to:

  • Rick & Morty characters (exercise 2)
  • Rick & Morty episodes (exercise 6)
  • Websocket (exercise 9)

Feel free to design this entry view as you wish.

Exercise 2 (view character list):

Add a view to show a list of characters. This must have those columns: Image, Name, Status, Species, and number of episodes (counter). Allow order by name, status, species, and number of episodes.

You must request the method GET https://rickandmortyapi.com/api/character that retrieves 20 characters.

Exercise 3:

Add the infinite scroll to load new content is continuously loaded and displayed as the user scrolls down the page (next page to the character API).

GET https://rickandmortyapi.com/api/character?page=2

Exercise 4:

Add a filter to the list. We want to filter the data in the list by name, status (dropdown), and species. Add a button to clear the filter and disable the filter.

Exercise 5 (view character detail):

Add a new component to get details about the selected character. Try to show all the images. Design this view as you wish.

GET https://rickandmortyapi.com/api/character/2

Exercise 6 (view episode list):

A new view will show a list of episodes, ordered by season and episode number (ascending). The list will contains: Season, Episode number, name and the air date. When the user clicks on an episode, he will navigate to the episode detail view (next exercise).

GET https://rickandmortyapi.com/api/episode

Exercise 7 (view episode detail):

This view will show the details of the episode. Show as much information as possible, including the same list of characters (same list format of the exercise 2, without filter). You can also navigate from this list of character to the character detail.

GET https://rickandmortyapi.com/api/episode/28

Exercise 8 (print):

We would like to print those forms using a regular printer. What would be your suggestion? Try to add it to the episode detail view.

Exercise 9 (websocket view):

Create a websocket connection to wss://ws.postman-echo.com/raw. Send a json message like "message": "Hello there!" and write the socket response below.

Exercise 10 (SSO):

Add SSO to your app. You can use free providers for demo like this: SAML SSO provided by Oauth0: https://auth0.com/docs/authenticate/protocols/saml/saml-configuration/configure-auth0-as-service-and-identity-provider or other you prefer.
